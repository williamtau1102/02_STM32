|    文件夹名称    | 作用                                                                                                                                                                                                |
| :---------: | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
|     Doc     | 文档，便于项目维护。                                                                                                                                                                                        |
|   Driver    | 存放内核文件、启动文件、固件库、硬件相关的驱动层等文件。                                                                                                                                                                      |
| Middlewares | 存放第三方提供的中间层代码，如 USMART、MALLOC、TEXT、FATFS、USB、LWIP、各种 OS、各种 GUI 等。                                                                                                                                 |
|   Output    | 存放编译器编译工程输出的中间文件，比如 .hex、 .bin、 .o 文件等。MDK 软件通过魔术棒可设置该文件夹为 Output 路径后，Listings 和 Objects 文件夹及相关内容会移动到该文件夹下。                                                                                       |
|  Projects   | 存放编译器（MDK、 IAR 等） 工程文件。MDK 在该文件夹下默认自动创建 DebugConfig（存放设置信息文件，不可删除）、Listings（存放编译过程中产生的链接列表等文件） 和 Objects（存放编译过程中产生的调试信息、hex 文件） 三个文件夹。为了便于管理，会把输出在 Listings 和 Objects 文件夹的内容，统一改为输出到 Output 文件夹中。 |
|    User     | 存放用户编写的代码，如 main.c、APP 等。                                                                                                                                                                         |
|  Readme.md  | 介绍项目的基本功能、版本迭代记录等内容。                                                                                                                                                                              |
